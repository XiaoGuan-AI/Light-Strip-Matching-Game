# AI元器件分拣箱 - 可扩展架构设计

## 核心挑战

- 元器件数量: 100+ 个
- LED数量: 50 个（可扩展）
- 需要: 灵活映射机制

## 解决方案: 分组溢出机制

### LED显示策略

```
元器件库 (100+)
  |
  |-- C001 ~ C050  --> LED[0] ~ LED[49]  (第1组)
  |
  |-- C051 ~ C100  --> LED[0] ~ LED[49] + 溢出标记 (第2组)
  |
  |-- C101 ~ C150  --> LED[0] ~ LED[49] + 溢出标记 (第3组)

溢出处理:
  - 找到C051时: LED[0]闪烁 + 喇叭响2次
  - 找到C101时: LED[0]闪烁 + 喇叭响3次
```

## 核心设计

### 1. component_config.h - 可配置参数

```cpp
#define LED_STRIP_PIN       GPIO_NUM_18
#define LED_STRIP_LENGTH    50      // 可改为64/100/256

#define BUZZER_PIN          GPIO_NUM_21

#define STATIC_IP           "192.168.3.238"

#define MAX_COMPONENTS      500     // 最大支持500个
```

### 2. component_database - 动态NVS存储

- NVS分区存储元器件列表
- 支持运行时添加/删除
- Web界面管理元器件

### 3. 分组溢出逻辑

```cpp
// 查找时计算分组
uint8_t actual_led = component.led_index % LED_STRIP_LENGTH;
uint8_t group = component.led_index / LED_STRIP_LENGTH + 1;

// LED闪烁 + 喇叭按组别次数响
led_control.Highlight(actual_led, r, g, b);
for (int i = 0; i < group; i++) {
    buzzer.Beep(100ms);
    vTaskDelay(150ms);
}
```
