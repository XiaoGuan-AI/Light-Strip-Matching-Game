# AI元器件分拣箱 - 实时扫码录入方案

## 一、功能概述

当前扫码入口以“微信式实时扫码”作为主流程：

| 方式 | 适用场景 | 行为 |
|------|----------|------|
| 📷 实时扫码 | 微信内或手机浏览器可扫码时 | 微信内优先拉起原生扫一扫；其他浏览器连续识别二维码，提取编号后立即写入并绑定 LED |
| 🖼 拍照识别 | 摄像头实时扫码不稳定时 | 选择或拍摄图片，识别成功后立即写入 |
| ✏️ 手动输入 | 无法扫码或需要补录时 | 输入编号后立即写入 |

界面中不再以“先收集、后统一导入”为主流程，扫码成功即完成录入，并在页面展示最近记录。

---

## 二、使用方法

### 2.1 访问界面

1. 手机连接 ESP32 的 WiFi 热点。
2. 打开浏览器访问：`http://192.168.3.75:32323/`

主要入口如下：

```text
┌─────────────────────────────────────────┐
│          AI元器件分拣箱                  │
├─────────────────────────────────────────┤
│  🔍 查找元器件                           │
│  [输入元器件编号... ] [查找]             │
├─────────────────────────────────────────┤
│  📷 实时扫码录入                         │
│  [📷 开始扫码] [🖼 拍照识别（备用）]      │
│                                         │
│  最近记录 / 已录入 / 已存在              │
│  [🧹 清空记录]                           │
│                                         │
│  或手动输入元器件编号立即录入            │
│  [输入元器件编号 ] [立即添加]            │
├─────────────────────────────────────────┤
│  📦 元器件列表                           │
└─────────────────────────────────────────┘
```

### 2.2 实时扫码流程

```text
1. 点击「📷 开始扫码」
2. 如果在微信内打开，页面优先拉起微信原生扫一扫
3. 如果不在微信内，则请求浏览器摄像头权限并进入实时扫描
4. 对准元器件袋子上的二维码
5. 页面自动提取二维码中的 `pc` / `part_number` / `pn`
6. 识别成功后立即调用 `/api/components/add`
7. 设备立即分配空闲逻辑槽位并绑定 LED
8. 页面将结果加入“最近记录”，无需额外确认导入
```

说明：

- 为避免同一二维码在连续扫码中被重复写入，前端会对短时间内的相同扫码结果做抑制，微信原生扫一扫与浏览器摄像头扫码都会复用这套去重逻辑。
- “清空记录”会同时清空最近历史、计数器和前端扫码去重状态，便于重新开始一轮录入。
- 手动输入和拍照识别属于备用路径，不应被相机最近扫描去重拦截。

#### 二维码内容要求

支持以下情况：

```text
{on:SO25053021861,pc:C2930027,pm:FRC0603J103 TS,...}
```

系统会优先提取以下字段中的编号：

- `pc:`
- `part_number:`
- `part-number:`
- `pn:`

如果二维码内容本身就是合法编号，也可直接识别。

### 2.3 拍照识别流程

```text
1. 点击「🖼 拍照识别（备用）」
2. 拍摄或选择图片
3. 页面在本地尝试识别二维码
4. 识别成功后立即录入并更新最近记录
```

适合以下情况：

- 浏览器无法稳定打开实时摄像头
- 现场光线较差，需要先拍照再识别
- 单张图片补录

### 2.4 手动输入流程

```text
1. 在手动输入框中输入元器件编号
2. 点击「立即添加」或按回车
3. 页面立即调用 `/api/components/add`
4. 成功后直接分配 LED 并写入列表
```

手动输入仅作为补录手段，但仍然走“立即录入”流程，而不是排队等待统一导入。

## 三、功能说明

### 3.1 自动绑定规则

| 规则 | 说明 |
|------|------|
| LED 分配 | 依次分配到下一个空闲逻辑槽位 |
| 分组计算 | `actual_led_index = logical_led_index % 50`，超过 50 自动进入下一组 |
| 颜色分配 | 使用调色板为逻辑槽位分配固定颜色，便于识别 |

### 3.2 录入反馈

- 新编号：立即写入数据库，并让对应 LED 快速闪烁 3 次。
- 已存在：不重复创建，返回原有 LED 位置，并闪烁已有位置提示。
- 失败：页面显示错误信息，不写入数据库。

### 3.3 最近记录

页面会维护最近 20 条识别/录入结果，显示：

- 编号
- 来源：实时扫码 / 拍照识别 / 手动输入
- 状态：已录入 / 已存在 / 失败
- LED 位置与组号

“清空记录”只重置前端扫码会话状态，不会删除已经保存到数据库的元器件数据。

## 四、摄像头问题排查

### 常见问题

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `navigator.mediaDevices undefined` | 浏览器不支持 | 使用 Chrome / Edge / Safari |
| `NotAllowedError` | 权限被拒绝 | 点击地址栏锁图标并允许摄像头 |
| `NotFoundError` | 未找到摄像头 | 检查设备摄像头是否可用 |
| `NotReadableError` | 摄像头被占用 | 关闭其他占用摄像头的应用 |

### 兼容浏览器

- Chrome（推荐）
- Edge
- Safari（iOS）
- Firefox（可能需要额外授权）

### 备用方案

如果实时扫码不可用，请按以下顺序降级：

1. 使用“拍照识别（备用）”
2. 如果图片也无法识别，再使用“手动输入立即添加”

## 五、API接口说明

### 5.1 立即录入元器件

**接口**: `POST /api/components/add`

**请求体**:

```json
{
  "part_number": "C2930027",
  "scan_text": "{on:SO25053021861,pc:C2930027,pm:FRC0603J103 TS,...}"
}
```

说明：

- `part_number` 可直接传入编号。
- `scan_text` 可传入原始二维码内容，服务端会再次尝试提取编号。

**成功响应示例**:

```json
{
  "success": true,
  "duplicate": false,
  "part_number": "C2930027",
  "led_index": 5,
  "actual_led_index": 5,
  "group": 1,
  "r": 120,
  "g": 200,
  "b": 80,
  "message": "Added successfully"
}
```

**重复响应示例**:

```json
{
  "success": false,
  "duplicate": true,
  "part_number": "C2930027",
  "led_index": 5,
  "actual_led_index": 5,
  "group": 1,
  "r": 120,
  "g": 200,
  "b": 80,
  "error": "编号已存在，请勿重复添加"
}
```

### 5.2 兼容接口：批量添加

**接口**: `POST /api/components/batch_add`

该接口仍可保留给兼容性或脚本调用使用，但不再是 Web UI 主流程。当前页面默认逐条立即录入，不依赖该接口。

## 六、实现说明

### 6.1 涉及文件

| 文件 | 说明 |
|------|------|
| `main/iot/component_http_server.cc` | 实时扫码页面、图片识别、手动立即录入、最近记录与前端去重逻辑 |
| `main/iot/component_sorter.cc` | `/api/components/add` 立即写入逻辑，以及兼容性的 `/api/components/batch_add` |
| `main/iot/component_database.cc` | 空闲逻辑槽位分配与持久化 |

### 6.2 主流程

```text
扫码/拍照/手动输入
    -> 提取 part_number
    -> POST /api/components/add
    -> 服务端校验是否已存在
    -> 新编号则分配逻辑槽位与颜色
    -> 返回 LED 位置 / 分组 / 颜色
    -> 页面更新最近记录与提示
```
